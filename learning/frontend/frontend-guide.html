SECTION 01

Frontend Architecture

The React frontend acts as the user-facing layer of your AI SaaS. It captures input, sends requests to the FastAPI backend, and renders streaming AI responses — all with zero coupling to the AI model itself.

Key Design Decision: The frontend never talks directly to the LLM. All AI logic lives in FastAPI — React only needs to know the POST /ask contract. This keeps the frontend clean and swappable.

Data Flow (per message)

01

User types + submits

ChatInput captures the message. handleSubmit fires, calling the custom hook useSendMessage().

02

Optimistic update

The user message is immediately added to Zustand store so the UI feels instant. A loading bubble appears.

03

API call via api.js

Axios POSTs { message, chat_id } to VITE_API_URL/ask. Error handling and timeout logic lives here.

04

Response → Store → UI

FastAPI returns { response: "..." }. Zustand replaces the loading bubble with the real response. ChatWindow auto-scrolls.

SECTION 03

Chat Components

Three core components build the chat UI. Each is isolated, testable, and uses Tailwind for styling.

💬 ChatWindow.jsx

Renders the scrollable message list. Auto-scrolls to the bottom on new messages using a ref.

✏️ ChatInput.jsx

Text area + send button. Handles Enter key, disabled state during loading, and focus management.

🫧 MessageBubble.jsx

Renders a single message. Conditionally styled for user (right, purple) vs AI (left, dark). Supports Markdown.

⏳ TypingIndicator.jsx

Three animated dots shown while waiting for the AI response. Signals loading without a spinner.

ChatWindow.jsx

components/ChatWindow.jsx 
import { useEffect, useRef } from 'react'
import { useChatStore } from '../store/chatStore'
import MessageBubble from './MessageBubble'
import TypingIndicator from './TypingIndicator'

export default function ChatWindow() {
  const { messages, isLoading } = useChatStore()
  const bottomRef = useRef(null)

  // Auto-scroll to bottom whenever messages update
  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages, isLoading])

  return (
    <div className="flex-1 overflow-y-auto px-4 py-6 space-y-4">
      {messages.map((msg, i) => (
        <MessageBubble key={i} message={msg} />
      ))}

      {/* Show typing dots while waiting for response */}
      {isLoading && <TypingIndicator />}

      {/* Invisible anchor for auto-scroll */}
      <div ref={bottomRef} />
    </div>
  )
}

MessageBubble.jsx

components/MessageBubble.jsx 
import ReactMarkdown from 'react-markdown'
import { motion } from 'framer-motion'

export default function MessageBubble({ message }) {
  const isUser = message.role === 'user'

  return (
    <motion.div
      initial={{ opacity: 0, y: 8 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.2 }}
      className={`flex ${isUser ? 'justify-end' : 'justify-start'}`}
    >
      <div className={`
        max-w-[75%] px-4 py-3 rounded-2xl text-sm leading-relaxed
        ${isUser
          ? 'bg-purple-600/80 text-white rounded-br-sm'
          : 'bg-[#181a24] border border-[#252840] text-[#cdd9e5] rounded-bl-sm'
        }
      `}>
        {isUser ? (
          {/* User messages: plain text */}
          <p>{message.content}</p>
        ) : (
          {/* AI messages: render Markdown */}
          <ReactMarkdown
            components={{
              code: ({node, ...props}) =>
                <code className="bg-black/40 px-1.5 py-0.5 rounded text-purple-300 text-xs font-mono" {...props} />
            }}
          >
            {message.content}
          </ReactMarkdown>
        )}
      </div>
    </motion.div>
  )
}

ChatInput.jsx

components/ChatInput.jsx 
import { useState } from 'react'
import { useSendMessage } from '../hooks/useSendMessage'
import { useChatStore } from '../store/chatStore'

export default function ChatInput() {
  const [input, setInput] = useState('')
  const { sendMessage } = useSendMessage()
  const { isLoading } = useChatStore()

  const handleSubmit = async () => {
    if (!input.trim() || isLoading) return
    const msg = input.trim()
    setInput('')          // clear input immediately
    await sendMessage(msg)
  }

  const handleKeyDown = (e) => {
    // Send on Enter, allow Shift+Enter for newlines
    if (e.key === 'Enter' && !e.shiftKey) {
      e.preventDefault()
      handleSubmit()
    }
  }

  return (
    <div className="px-4 py-4 border-t border-[#1e2030] bg-[#0c0e14]">
      <div className="flex items-end gap-3 bg-[#12141c] border border-[#252840] rounded-2xl px-4 py-3">
        <textarea
          value={input}
          onChange={(e) => setInput(e.target.value)}
          onKeyDown={handleKeyDown}
          placeholder="Ask anything..."
          rows={1}
          disabled={isLoading}
          className="flex-1 bg-transparent resize-none text-sm text-[#cdd9e5] placeholder-[#6e7799] outline-none"
        />
        <button
          onClick={handleSubmit}
          disabled={!input.trim() || isLoading}
          className="px-4 py-2 bg-purple-600 hover:bg-purple-500 disabled:opacity-40 text-white text-xs font-mono rounded-xl transition-all"
        >
          {isLoading ? '...' : 'Send ↑'}
        </button>
      </div>
      <p className="text-[0.65rem] text-[#6e7799] mt-2 pl-1 font-mono">
        Enter to send · Shift+Enter for newline
      </p>
    </div>
  )
}

SECTION 04

API Integration Layer

All API communication is centralized in services/api.js. Never call fetch or Axios directly from components — always go through this layer for maintainability.

Axios Instance (services/api.js)

services/api.js 
import axios from 'axios'

// Single Axios instance — base URL comes from env
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:8000',
  timeout: 30000,   // 30s — LLMs can be slow
  headers: { 'Content-Type': 'application/json' },
})

/**
 * Send a message to the RAG chatbot backend.
 * @param {string} message   - user's text
 * @param {string} chat_id   - session identifier
 * @returns {Promise<string>} - AI response text
 */
export async function askBot(message, chat_id) {
  const { data } = await api.post('/ask', { message, chat_id })
  return data.response
}

// Re-export instance for future endpoints
export default api

Custom Hook (hooks/useSendMessage.js)

hooks/useSendMessage.js 
import { askBot } from '../services/api'
import { useChatStore } from '../store/chatStore'

export function useSendMessage() {
  const { addMessage, setLoading, setError, activeChatId } = useChatStore()

  const sendMessage = async (text) => {
    // 1. Add user message optimistically
    addMessage({ role: 'user', content: text })
    setLoading(true)
    setError(null)

    try {
      // 2. Call API
      const response = await askBot(text, activeChatId)

      // 3. Add AI response to store
      addMessage({ role: 'assistant', content: response })
    } catch (err) {
      // 4. Show error message in chat instead of crashing
      const errMsg = err.response?.data?.detail || 'Something went wrong. Try again.'
      addMessage({ role: 'assistant', content: `⚠️ ${errMsg}`, isError: true })
      setError(errMsg)
    } finally {
      setLoading(false)
    }
  }

  return { sendMessage }
}
⚡

Error UX: Never let API errors silently fail or crash the UI. The pattern above adds the error as a chat bubble — users know something went wrong without losing their conversation context.

SECTION 05

State Management

Zustand replaces both Redux and Context API for this use case. It's minimal, fast, and requires no providers or boilerplate.

Chat Store (store/chatStore.js)

store/chatStore.js 
import { create } from 'zustand'
import { persist } from 'zustand/middleware'
import { v4 as uuidv4 } from 'uuid'

export const useChatStore = create(
  persist(
    (set, get) => ({
      // State
      messages: [],              // { role, content, isError? }[]
      isLoading: false,
      error: null,
      activeChatId: uuidv4(),   // unique per browser session

      // Actions
      addMessage: (msg) =>
        set((state) => ({ messages: [...state.messages, msg] })),

      setLoading: (val) => set({ isLoading: val }),
      setError:   (val) => set({ error: val }),

      clearChat: () => set({
        messages: [],
        activeChatId: uuidv4(),  // fresh session ID
      }),
    }),
    {
      name: 'ai-chat-storage',   // persists to localStorage
      partialize: (state) => ({  // only persist messages + chatId
        messages: state.messages,
        activeChatId: state.activeChatId,
      }),
    }
  )
)
✅

The persist middleware saves messages to localStorage automatically — users won't lose their chat on page refresh. Use partialize to exclude transient state like isLoading.

State Shape

▸ Zustand Store Structure
useChatStore {
  messages: [
    { role: 'user',      content: 'What is RAG?' },
    { role: 'assistant', content: 'RAG stands for...' },
    { role: 'assistant', content: '⚠️ Error...', isError: true }
  ],
  isLoading:    boolean    ← true while waiting for API
  error:        string|null
  activeChatId: uuid       ← sent with every request
}

SECTION 08

Deployment

Deploy the React frontend to Vercel (zero-config for Vite). Deploy the FastAPI backend separately on Railway or Render.

Frontend → Vercel

01

Push to GitHub

Vercel connects to your repo. Every push to main auto-deploys. Create the repo and push your code.

02

Import on Vercel

Go to vercel.com → New Project → Import your repo. Vercel auto-detects Vite and sets build command to npm run build, output to dist.

03

Set Environment Variables

In Vercel dashboard → Settings → Environment Variables. Add VITE_API_URL pointing to your deployed backend.

04

Deploy

Trigger a redeploy after setting env vars. Your chat UI is now live on a *.vercel.app domain.

Backend → Railway (FastAPI)

Procfile (add to FastAPI repo root) 
web: uvicorn main:app --host 0.0.0.0 --port $PORT
⚠️

CORS: Your deployed FastAPI must allow requests from your Vercel domain. Add this to your FastAPI app:

main.py — CORS middleware 
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",           # Vite dev
        "https://your-app.vercel.app",       # Production
    ],
    allow_methods=["*"],
    allow_headers=["*"],
)
SECTION 09

Advanced Features

Once the core chat is working, these features push it toward a real product.

Streaming Responses

If your FastAPI backend supports StreamingResponse, you can render tokens as they arrive — like ChatGPT. This requires switching from Axios to fetch with the ReadableStream API:

services/api.js — streaming version 
export async function askBotStream(message, chat_id, onChunk) {
  const res = await fetch(`${import.meta.env.VITE_API_URL}/ask/stream`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ message, chat_id })
  })

  const reader = res.body.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const chunk = decoder.decode(value)
    onChunk(chunk)   // callback updates store chunk-by-chunk
  }
}

Multi-Chat Sidebar

Extend the store to hold multiple chat sessions. Add a chats array where each item has an id, title, and messages. The sidebar renders this list and switching chats updates activeChatId.
