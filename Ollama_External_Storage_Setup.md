# Ollama External Storage Setup Guide

This document explains the steps taken to redirect Ollama model storage from the default macOS location to an external drive (`/Volumes/MacDisk_1TB`).

## Context
Ollama defaults to storing models in `~/.ollama/models`. Large models (like `sorc/qwen3.5-claude-4.6-opus`, which is ~10GB) can quickly exhaust internal disk space. We moved the storage to an external volume to preserve primary drive capacity.

## Implementation: The Symbolic Link Method
The most robust way to handle this on macOS (compatible with both the CLI and the Ollama background app) is using a **Symbolic Link**.

### Step 1: Create the External Directory
We created a dedicated folder on the external drive to hold the model data:
```bash
mkdir -p "/Volumes/MacDisk_1TB/ollama_models"
```

### Step 2: Redirect the Default Path
We removed the existing default models directory and replaced it with a link pointing to the external folder:
```bash
# Remove existing local models folder (Note: this removes models stored on the internal drive)
rm -rf ~/.ollama/models

# Create the symbolic link
ln -s "/Volumes/MacDisk_1TB/ollama_models" ~/.ollama/models
```

## Alternative: Environment Variable Method
If you only use Ollama via the terminal and do *not* want to use a symbolic link, you can set an environment variable in your shell profile (`~/.zshrc` or `~/.bash_profile`):

```bash
export OLLAMA_MODELS="/Volumes/MacDisk_1TB/ollama_models"
```

*Note: This method is less recommended for general use because the Ollama macOS GUI application may not honor variables set in your shell profile unless the app is launched from that specific terminal session.*

## Verification
To verify the setup, you can run:
```bash
ls -ld ~/.ollama/models
```
The output should show an arrow pointing to the external path:
`lrwxr-xr-x ... /Users/yourname/.ollama/models -> /Volumes/MacDisk_1TB/ollama_models`
